# V2.0 基线审计 — 设计文档

**日期**: 2026-06-18
**状态**: 待实施
**类型**: 数据收集（不修改业务代码）

## 1. 目标

运行现有 V1 二进制，采集至少 10 分钟推流数据，生成基线报告，作为 V2 优化的对照基准。

## 2. 范围

### 纳入

- 环境信息记录（OS、CPU、分辨率、RTSP server、FFmpeg/SDL 版本）
- 10 分钟推流（视频 + 音频），使用当前默认编码参数
- CSV 数据收集（现有 14 字段）
- 日志中 VBV underflow 及其他异常的统计
- 可接受/不可接受项分类

### 排除

- 不修改任何 .cpp/.h 文件
- 不新增统计字段
- 不调整编码参数

## 5. 运行命令

```
RTSP-Pusher.exe \
  --rtsp-url rtsp://192.168.42.116:25544/test \
  --stats-csv csv_baseline.csv \
  --log log_baseline.log
```

音频默认启用（`enableAudio = true`），无需额外参数。

## 6. 收集指标

从 CSV 和日志提取：

| 指标 | 来源 | 统计方式 |
|------|------|----------|
| 视频采集 fps | CSV videoCaptured 差值 / 5s | 全时段平均 + 稳定段平均 |
| 视频编码 fps | CSV videoEncoded 差值 / 5s | 同上 |
| 丢帧数 | CSV videoDropped 终值 | 累计 |
| 音频采集字节 | CSV audioBytes 终值 | 累计 |
| AAC 编码帧数 | CSV audioEncoded 终值 | 累计 |
| RTSP 写包数 | CSV packetsWritten 终值 | 累计 |
| 写错误数 | CSV writeErrors 终值 | 累计 |
| 重连次数 | CSV reconnects 终值 | 累计 |
| rawQueue max | CSV rawQueue 最大值 | 全时段 |
| encQueue max | CSV encQueue 最大值 | 全时段 |
| encodeMaxUs | CSV encodeMaxUs 最大值 | 全时段 |
| muxWriteMaxUs | CSV muxWriteMaxUs 最大值 | 全时段 |
| VBV underflow | 日志 grep | 次数 + 首次/末次帧号 |
| bitrateKbps | CSV | 应为 0（已知问题，记录为可接受或不可接受） |

## 7. 产出物

`docs/v2-baseline-report-2026-06-18.md`，格式：

```
# V2 基线审计报告

## 环境信息
## 测试参数
## 运行结果
  - 时长
  - 各指标汇总表
  - VBV underflow 详细
  - 日志关键事件时序
## 可接受项
## 不可接受项
## 结论
```

## 8. 验收标准

- [ ] 推流持续 ≥ 10 分钟
- [ ] CSV 文件包含 ≥ 120 行数据
- [ ] 日志文件完整无截断
- [ ] 报告列出至少 3 个可接受项和 3 个不可接受项
- [ ] 报告中所有指标有具体数值，无 "N/A" 或空值
- [ ] 播放端可正常观看（主观确认）
